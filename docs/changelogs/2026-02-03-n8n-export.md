# 2026-02-03 – n8n export

## Sammanfattning
Export från n8n‑instansen via API (localhost). Följande workflows uppdaterades i `n8n/flows/`.

## Ändringar per workflow
- **FGC THN - Checkin Orchestrator v4**
  - `Merge Results`: `member`, `startgg` och `is_guest` bygger nu på faktiska API‑resultat (t.ex. `isActualMember` / `isActualStartgg`) i stället för “check passed”. Status/missing använder fortfarande krav‑logiken.
  - Airtable‑credentials bytta.
- **FGC THN - Start.gg Check**
  - Inga tydliga funktionsändringar; diffen ser ut att vara metadata (version/IDs/pinData).
- **FGC THN - eBas Membership Check**
  - Ny “Skip eBas”‑gren som returnerar `false` tidigt när personnummer saknas.
  - Flera metadata/notes‑justeringar.
- **FGC THN - eBas Register**
  - Workflow markerad som `active: false`.
  - Airtable‑lookup ändrad till list‑baserad base/table‑referens.
  - `filterByFormula` i “Find Player in Airtable” ser ut att ha ändrats (bör verifieras att uttrycket är korrekt).
  - `Update Member Status`: `id`‑fält borttaget.

## Noteringar
- Rekommenderas att dubbelkolla `filterByFormula` i eBas Register för korrekt syntax.
